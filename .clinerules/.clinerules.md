# Cline Rules — Generic (applies to all projects)

## Before You Code
- Before writing any code, state your interpretation of the request in one sentence.
- If anything is ambiguous, ask before proceeding — do not guess and do not assume.

## Minimal Footprint
- Write the minimum code that satisfies the request. Nothing more.
- Do not create abstractions, helpers, utilities, or extra layers unless explicitly asked.
- Do not add new dependencies without asking first.
- Do not implement features beyond what was asked. If you think something additional is needed, ask rather than build it.

## Surgical Changes
- Only modify code directly related to the request.
- Do not refactor, rename, reformat, or clean up surrounding code unless explicitly asked.
- Every changed line must trace directly back to the request.

## General
- Write self-documenting code. Do not add comments that explain what the code does.
- Keep functions small and single-purpose.
- If the user rejects an edit, stop immediately and ask what to do next.

## Git
- Do not create a git commit automatically. Always ask the user before committing.
- Use conventional commit format: `type(scope): message` (e.g. `feat(auth): add login endpoint`).
- Never use `--no-verify` or bypass git hooks.
